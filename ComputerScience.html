Object Oriented Programming





There are 4 main pillars of Object Oriented programming:

  1. Encapsulation

    2. ENCAPSULATION: This is a way of hiding data that you don't want the user or any other programmer modifying. For instance, a particular variable can be encapsulated which would not allow the user or any other programmer working on the same program to change or alter the nature/value/name of the variable. In order to accomplish Encapsulation, access modifiers are used. Example of access modifiers are private, protected, and public. When a variable is set as private, one cannot access that variable unless it is in the same class. If one has to access that particular variable in a separate class, a getter and setter method has to be made in the class in which the variable is declared. Then, the getter and setter methods are called in the class where the variable is to be used. A getter method "gets" the value stored in the variable. A setter method stores/"sets" a value in the variable.

  2. Abstraction
  3. Inheritance
  4. Polymorphism


Hospital Management System

The Hospital Management system is a system that keeps track of a variety of aspects of a hospital.
It allows the user to view information about the Hospital:

  1. Get access to Hospital related information:
    1. Name of Hospital
    2. Contact information or phone number of the hospital
    3. The address of the Hospital


  2. Hospital Staff
    1. Adding new Staff
    2. See the list of the staff working at the Hospital:
      1. ID number
      2. First Name
      3. Last Name


  3. Doctor information:
    1. Adding new doctor to the Hospital or the Hospital Management System
    2. Getting information about a particular doctor:
      1. ID number
      2. First Name
      3. Last Name
      4. Type of doctor
      5. Whether or not they have appointments available


  4. Patient Information
    1. Adding new patient
    2. Assigning patients to a particular Doctor
    3. See the list of patients that a particular Doctor looks after:
      1. ID number
      2. First Name
      3. Last Name
      4. Contact information or phone number
      5. Address of the patient
    4. Checking appointments for a particular patient of a particular doctor.


  5. Nurse Information:
    1. Adding new nurse to the Hospital or the Hospital Management System
    2. Finding information about a particular nurse:
      1. ID number
      2. First Name
      3. Last Name
      4. The ID, First and Last name of the Doctor they are assigned to.
    3. Assigning nurses to different doctors.


  6. Maintaining patient Records
    1. Adding diagnosis and age of of the patient to the patient record.
    2. Finding all information written in a record about a patient:
      1. ID number
      2. First Name
      3. Last Name
      4. Phone Number
      5. Address
      6. Diagnosis
      7. Age
      8. The doctor they are assigned to.
    3. Assigning the patient a family Doctor
    4. Changing the diagnosis of the patient.

Features used in making the Hospital Management System:

  1. Inheritance

    2. Inheritance is when a class derives attributes and methods from another class. The class it derives from is called the parent class, and the class which derives the information is called the child class.
    Inheritance is defined using the extend keyword. Inheritance explains a is-a relationship between the classes.

  2. Composition

    3. Composition explains a has-a relationship between classes. It uses instance variables of other classes. In other words, when declaring a variable, the “type” can include int, String, double, etc. In the case of composition, the type can include the names of other classes. The variable then attains all the features of the class which is its “type”.

  3. Aggregation

    4. This is when two or more objects are related but can function without the existence of one another and without either one of them being the owner. For instance, a doctor can have multiple patients, likewise, a patient can visit multiple doctors.
